### 玩家接口设计:

一个玩家对应一个goroutine, 每个goroutine接收信息类型：     
1. 来自客户端的    
2. 来自其他玩家的    
3. 时钟信号

### 持久化
玩家在线时agent负责变更内存中的升级信息, 并将升级结果持久化到数据库，   
在线时，升级逻辑以agent为准, 即直接改变数据库中的数据.    
事件服务器通过CAS的方式变更key:value, 保证玩家在线时逻辑处理的一致性.     
事件服务器的策略是, 在满足条件的时候执行，或者忽略.
